# Roadmap

## 0.1 — SDK core and mandatory registration scenario

- [x] Async HTTP transport
- [x] Access-token manager and cache
- [x] Generic OpenAPI requests
- [x] Multipart and byte responses
- [x] Event verification/decryption
- [x] One-click app registration
- [x] `create-an-app-in-one-click-rust`
- [x] Representative typed IM API

## 0.2 — Generated high-frequency services

- [ ] Generate request/response/resource modules from platform metadata
- [ ] IM, Contact, Drive, Docx, Bitable, Calendar, Wiki, Sheets
- [ ] Pagination helpers
- [ ] Upload/download helpers with streaming files
- [ ] Endpoint-level access-token validation

## 0.3 — Long connections and advanced auth

- [ ] WebSocket event client
- [ ] Reconnect, heartbeat and backpressure
- [ ] Client assertion OAuth
- [ ] App ticket manager for marketplace callbacks
- [ ] External distributed cache examples

## 1.0 — Stable generated surface

- [ ] Full supported OpenAPI service set
- [ ] SemVer compatibility policy
- [ ] crates.io publication and docs.rs coverage
- [ ] Release automation and changelog generation
